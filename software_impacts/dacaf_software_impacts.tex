\noindent
\textbf{Abstract}\\
In multi-label classification (MLC), different evaluation metrics call for different
Bayes-Optimal Predictions (BOPs), and no single model is optimal for all of them.
Probabilistic MLC allows the target metric to be optimized during the prediction phase,
but computing the exact BOP is intractable in general, and existing tools are
limited---covering only a few metrics, relying on strong assumptions such as label
independence, or lacking a reusable, well-tested implementation. We present DaCaF, a
Python package implementing the Divide-and-Conquer and Fusion approach that, given any
probabilistic model $P(y\mid x)$, returns the exact BOP for a chosen metric, covering two
families of MLC metrics. The package provides seven per-metric prediction rules, each
verified against brute-force enumeration, together with a command-line interface, a
library API, and reproducible evaluation scripts.

\section{Motivation and significance}\label{sec:motivation}

Multi-label classification (MLC) is a setting in which each instance can belong to
any subset of the labels, with applications ranging from text and image annotation to
high-stakes domains such as clinical decision-making. To compare ground-truth label sets
and predictions, a variety of evaluation metrics with different characteristics have been
proposed. Probabilistic MLC offers unique advantages by allowing the optimization of
different metrics during the prediction phase: once a probabilistic classifier estimates
the conditional probability of each labeling, the Bayes-Optimal Prediction (BOP) of a
target metric---the prediction $\hat{y}$ that maximizes the expected score of that
metric---can be sought. As different metrics may call for significantly different BOPs, a
mismatch between the target metric, i.e., the metric optimized during the prediction
phase, and the evaluation metric, i.e., the metric used to assess the classifier, can
negatively impact predictive performance.

However, computing the BOP in a naive way---by searching over the $2^L$ candidate
predictions for $L$ labels---quickly becomes intractable as the number of labels grows.
Existing studies are arguably limited in two respects: some focus on only a small number
of metrics, such as subset 0/1 accuracy, Hamming accuracy, and $F_\beta$
\citep{pcc2010,fmax2011,waegeman2014}, yet require a different, metric-specific algorithm
for each metric; others rely on strong assumptions, such as label independence
\citep{pillai2017}. As a consequence, no reusable, exact, and broadly applicable tool
exists for finding BOPs across the metrics commonly used in probabilistic MLC.

We present DaCaF, a software package that implements the Divide-and-Conquer and Fusion
(DaCaF) approach \citep{dacaf2026}: given any probabilistic model $P(y\mid x)$, it returns
the exact BOP for a chosen metric, covering two families of commonly used MLC metrics. It
proceeds in two steps---a divide-and-conquer strategy followed by a fusion strategy---to
find the prediction that is optimal for the selected metric. DaCaF thereby (a) optimizes
the metric one actually cares about, (b) enables studying the mismatch between the target
and evaluation metrics in an exact way, undisturbed by approximation, and (c) provides a
diagnostic that flags metrics whose BOP has a trivial or very restricted
structure---useful for assessing the practical value of emerging MLC metrics.

\section{Software description}\label{sec:description}

\subsection{Software architecture}
DaCaF is an installable Python package (\texttt{dacaf\_mlc}; pip / Docker, with a console
script \texttt{dacaf-mlc}). Its core class \texttt{ProbabilisticClassifierChainCustom}
(PCC) wraps any scikit-learn base estimator and estimates $P(y\mid x)$ via a classifier
chain. Every inference rule rests on two algorithmic building blocks:
\begin{itemize}
  \item \textit{Divide-and-conquer}: partition the $2^L$ predictions into $L{+}1$ groups
        by number of relevant labels; the best in each group is found by sorting labels by
        a per-metric score, and the global best is the best across groups.
  \item \textit{Fusion}: the scores require marginal/pairwise probabilities, estimated by
        fusing the chain's dependent binary classifiers via ancestral sampling.
\end{itemize}
Supporting modules include \texttt{evaluation\_metrics.py}, \texttt{metrics\_registry.py},
\texttt{pipeline.py} (train / k-fold / run\_single), \texttt{evaluate.py} (CLI),
\texttt{arff\_dataset.py} (MULAN ARFF + 10-fold CV), \texttt{datasets.py}, and
\texttt{chest\_xray\_dataset/} (NIH feature extractor, [image] extra). A vendored
\texttt{skmultiflow} ClassifierChain base keeps the install self-contained.

\subsection{Software functionalities}
\begin{itemize}
  \item Seven per-metric Bayes-optimal rules: \texttt{predict\_fmeasure},
        \texttt{predict\_hamming}, \texttt{predict\_markedness}, \texttt{predict\_precision},
        \texttt{predict\_npv}, \texttt{predict\_recall}, \texttt{predict\_subset}. Each
        returns the prediction maximising the expected value of its target metric
        (conventions in CONVENTIONS.md).
  \item Covers two whole metric families at once and flags when a metric's optimum is
        \emph{trivial} (a metric-choice diagnostic).
  \item Batched predictor: one \texttt{predict\_proba} per chain level instead of
        $N\cdot L\cdot 2^L$; verified numerically equal to brute-force enumeration.
  \item Reproducible evaluation: a CLI and scripts (\texttt{reproduce\_tabular.sh}, Slurm)
        over MULAN tabular datasets and an NIH ChestX-ray14 image dataset.
  \item Correctness harness: every rule checked against brute-force expected-metric
        enumeration on small cases; CI on Python 3.10/3.12.
\end{itemize}

\subsection{Illustrative example}
A minimal library usage fits the model once and then queries the BOP for any metric:
\begin{lstlisting}[language=Python,basicstyle=\scriptsize\ttfamily]
from sklearn.linear_model import LogisticRegression
from dacaf_mlc.probability_classifier_chains \
    import ProbabilisticClassifierChainCustom
from dacaf_mlc.evaluation_metrics import EvaluationMetrics as EM

pcc = ProbabilisticClassifierChainCustom(
    LogisticRegression(max_iter=10_000))
pcc.fit(X_train, Y_train)            # Y: (n, L) binary
y_f1  = pcc.predict_fmeasure(X_test, beta=1)  # BOP for F1
y_ham = pcc.predict_hamming(X_test)           # BOP for Hamming
print(EM.f_beta(Y_test, y_f1), EM.hamming(Y_test, y_ham))
\end{lstlisting}

\section{Impact}\label{sec:impact}
DaCaF enables an exact research question that approximation would otherwise blur: how much
does optimising the wrong metric cost? Because the package is model-agnostic---it works
with \emph{any} probabilistic MLC model as a drop-in BOP layer on top of existing
$P(y\mid x)$ estimators---this question can be studied across estimators and datasets
without changing the inference code. Its breadth is unusual: a single tool covers two
whole metric families, and the trivial-optimum diagnostic guides metric selection by
flagging metrics whose Bayes-optimal prediction collapses to a degenerate form (as NPV and
Recall do in Table~\ref{tab:chd49}).

The software is designed for reuse beyond the companion paper. Its closed-form rules are
verified against brute-force enumeration and kept reproducible by CI, so results can be
trusted and rebuilt. The barrier to entry is low: the package is pip-installable and
Dockerised, ships scripted reproduction and bundled MULAN datasets, and lets a newcomer
reproduce a paper table in minutes. A registry pattern makes it straightforward to add
metrics and datasets, while the vendored base estimator keeps installs self-contained.
Beneficiaries include MLC researchers studying metric behaviour, applied ML practitioners
choosing a deployment metric, and educators teaching Bayes-optimal decision-making in MLC;
an NIH ChestX-ray14 demonstration shows the same API applied to clinical/image MLC.

\section{Conclusions}\label{sec:conclusions}
DaCaF turns a proved theoretical recipe into a reusable, verified, and exact per-metric
inference tool for probabilistic MLC. Future work includes supporting more base models and
metric families, and adding optional approximate fast paths for very large label spaces.
